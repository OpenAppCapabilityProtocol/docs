**OACP is MCP for mobile apps.**

Apps describe what they can do in a JSON file. AI Assistant (Hark) discovers those capabilities. Voice commands become Android Intents. Everything runs on-device. No cloud.

[Hark](/docs/hark/overview) is the open-source assistant that brings OACP to life. The name means "to listen." It's also my name. Eight apps support it today, with more coming.

## How it works

```
App ships oacp.json      →  Assistant discovers via ContentProvider
User speaks a command    →  On-device AI matches to the right capability
Assistant fires Intent   →  App handles the action, returns a result
```

No server. No API key. Everything runs on the device.
