## `OacpResult` - factory methods

| Method | Description |
|--------|-------------|
| `OacpResult.success(message, data?)` | Action completed |
| `OacpResult.accepted(message?)` | Action acknowledged, processing |
| `OacpResult.started(message?)` | Processing began |
| `OacpResult.error(code, message, retryable?)` | Action failed |
| `OacpResult.cancelled(message?)` | Action cancelled |

## `OacpParams` - typed parameter access

| Method | Returns | Notes |
|--------|---------|-------|
| `getString(name)` | `String?` | Matches by suffix, coerces from other types |
| `getInt(name)` | `Int?` | Coerces from string |
| `getLong(name)` | `Long?` | Coerces from string |
| `getDouble(name)` | `Double?` | Coerces from string |
| `getBoolean(name)` | `Boolean?` | Coerces from `"true"`/`"false"` |

All getters return `null` when the parameter is absent. Use Kotlin's `?:` operator for defaults or `return OacpResult.error(...)` to reject the request.

## `OacpReceiver` - abstract receiver

Subclass and override the single `onAction` method:

```kotlin
abstract fun onAction(
    context: Context,
    action: String,
    params: OacpParams,
    requestId: String?
): OacpResult?
```

Return `null` for fire-and-forget actions where the assistant should not wait for a result. Return an `OacpResult` to have it broadcast back automatically using the supplied `requestId`.

The receiver automatically:

- Calls `goAsync()` so your work runs on a background thread without blocking the `onReceive` 10-second limit
- Wraps exceptions in `ERROR_INTERNAL` results
- Uses `requestId` to correlate responses with the original request

## `OacpActivity` - abstract activity

Subclass and override `onOacpAction` for foreground actions:

```kotlin
abstract fun onOacpAction(action: String, params: OacpParams)
```

Protected members:

| Member | Type | Description |
|--------|------|-------------|
| `requestId` | `String?` | Extracted from `org.oacp.extra.REQUEST_ID` |
| `capabilityId` | `String?` | Derived from action string suffix |
| `params` | `OacpParams` | Type-safe parameter accessor |
| `sendResult(result, targetPackage?)` | method | Broadcasts an `OacpResult` back to the assistant |

Handles both `onCreate` and `onNewIntent` for `singleTask` launch mode.

## `OacpEntitySource` - entity interface

Implement to serve dynamic entity data:

```kotlin
interface OacpEntitySource {
    fun queryEntities(context: Context, entityType: String, query: String?): List<OacpEntity>
}
```

Register with `OacpProvider.setEntitySource(source)` in your `Application.onCreate()`.

## `OacpEntity` - entity data class

```kotlin
data class OacpEntity(
    val id: String,
    val displayName: String,
    val aliases: List<String> = emptyList(),
    val description: String? = null,
    val keywords: List<String> = emptyList(),
    val metadata: Map<String, Any> = emptyMap()
)
```

## `OacpProvider`

Auto-registered via manifest merger at the authority `${applicationId}.oacp`. You never need to instantiate or subclass it directly. It:

- Serves `oacp.json` at `content://${applicationId}.oacp/manifest`
- Serves `OACP.md` at `content://${applicationId}.oacp/context`
- Serves entity data at `content://${applicationId}.oacp/entities/<type>?q=query`
- Substitutes `__APPLICATION_ID__` placeholders with the real package name at runtime, so the same manifest works for `debug`, `staging`, and `release` variants without modification

Static method: `OacpProvider.setEntitySource(source: OacpEntitySource?)` registers the entity handler.
